Course: Data Mining  ·  Earthquakes Linear Regression

Introduction

Earthquakes are pretty cool natural events, but they're also scary and devastating. Predicting their magnitude can help prepare for evacuations or rescue relief, or it could even help engineers design buildings looking at magnitude patterns. This data set is the Global Earthquake and Tsunami Risk Assessment Dataset from Kaggle, compiled by Ahmed Uzaki. It has data on 782 earthquakes, and not too many features as there are only 13. All of the data is complete and there is no missing values! The data spans across 22 years, but I think it's mostly big magnitude earthquakes.

What is Regression?

Regression is a supervised learning method, and there are different kinds of regression algorithms. Regression typically predicts a continuous numeric value. Basically if you have a function, you give input features, and use that to predict an output feature. In this project I use linear regression to try and predict magnitude. However, it's important to note that logistic regression is not regression, but is a classification method I typically use. Mostly by default. Anyways, we can measure regression performance using RMSE - Root Mean Squared Error, MAE - Mean Absolute Error, and MSE - Mean Squared Error. The math is confusing, but I do know that it has to do with least squares.

Data Pre-processing

This dataset consisted of 0 missing values, as it is fully complete. This is great, as it was literally ready for machine learning. I did not have to change anything, but I did drop some columns such as year as I don't think that relates to magnitudes.

Modeling

For experiment 1, I used linear regression and dropped columns like tsunami and years. Tsunami is an obvious non-predictor, as it is a result of earthquakes. At least that was my thinking.

We can see that mmi has the highest correlation, however all of these have very low correlation coefficient scores, co close to 0. Thus, for experiment 2, I decided to drop mmi and cdi, the highest from both, just to see.

Now I can see that things don't look that much better. I still get very low scores. Thus, I decided to scale the features and see if standardizing the data would make a difference.

Now we finally have a score that is a lot higher, arount 25% or 0.25. This means that there is some correlation to magnitude, although maybe not too heavy or too reliable. Looking at all of the experiments, the results were very lack luster and low coreelation yielding. However after standardizing the features, we can see that sig, or the event significance which is how big an earthquake is, the number of stations detecting it, how wide it tis felt, and how important it could be, is about 0.25. This means there is some correlation, but not super strong so not quite reliable, however can give some idea on the magnitude. However just like mmi, sig is something that can or is found after magnitude, thus not being a great predictor, but they do relate.

Impact & Conclusion

The impact of this shows how standardizing/scaling features can affect the coefficient values. More importantly, this shows how feature choosing affects the data, and how this could be interpreted incorrectly. Sig is NOT something that predicts magnitude, because real world use case would not be practical. Why? As said before, if sig is reported by stations and people (like mmi), then that number is usually found after the magnitude is already found! So unless it takes a long time to get the magnitude, it cannot be considered a predictor.

I learned the essence of linear regression, but I learned what I cannot use it for. I really wanted to predict tsunamis, but that is classification. Also, I learned that scaling gave way better results, but that could be bad if the interpretation and understanding is off, which in our case we took a look and understand that we cannot necessarily reliably use this feature as a predictor. I also implemented feature selection and removal, which definitely affected the data.
